- Currently we do a full JFA pass for each object. While that is expensive it is actually necessary.
	One idea to optimise could be to draw each silhouette, and then use edge detection (Sobel) to combine everything into one edge texture (using and writing depths to ensure 	render order is still correct). However, this would mean that for any one pixel, we only know the one object it is closest to. If we want to have transparent outlines that 	may overlap, then this doesn't work together, so we have to choose.
- Wobble becomes jagged if we try to make it too steep, but that is also not easily solvable, since it is a fundamental flaw of the resolution of the outline/silhouette. Since a thick outline essentially means we are blowing up the resolution, so if we want to have a thick wobbly outline, we need a very large resolution, which seems wasteful.
